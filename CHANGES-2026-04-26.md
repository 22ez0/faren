## 3. Backup do banco de produção

### 3.1 Backup feito agora
`backups/faren-db-20260426T103629Z.sql.gz` — **21 MB**, 13 tabelas, dados reais
(18 usuários, 18 perfis, 215 visualizações de perfil, 13 posts, 24 links, 11
seguidores, 17 likes em perfil, 13 likes em post, 6 comentários, 1 ticket).

> **Aviso técnico:** o `pg_dump` 16.x do nix do Replit não conecta no Postgres
> 18 do Render (server version mismatch). Solução: baixei `pg_dump 18.0` portável
> de https://github.com/theseus-rs/postgresql-binaries (12 MB, sem
> instalação). O workflow do GitHub instala `postgresql-client-18` direto da
> apt do PostgreSQL.

### 3.2 Scripts locais
- `scripts/db-backup.sh` — dump comprimido, retém os 30 mais recentes.
- `scripts/db-restore.sh` — restore com prompt de segurança (use `FORCE=1` para CI).

Uso local:
```bash
PROD_DATABASE_URL="postgres://..." ./scripts/db-backup.sh backups
TARGET_DATABASE_URL="postgres://localhost/faren" ./scripts/db-restore.sh backups/faren-db-XXXX.sql.gz
```

### 3.3 Workflow automático (`.github/workflows/db-backup.yml`)
- Roda **diariamente às 03:00 UTC (00:00 BRT)** e sob demanda
  (`workflow_dispatch`).
- Instala `postgresql-client-18`, executa `scripts/db-backup.sh`.
- Faz upload do `.sql.gz` como **artifact com retenção de 30 dias** (acessível
  na aba Actions). Custo: zero (grátis para repos privados até 500 MB de
  artifacts).

## 4. AÇÃO MANUAL PENDENTE (única coisa que API não faz)

### Chaves S3 do R2
A API da Cloudflare **não** expõe criação de chaves S3 do R2 — só o dashboard.
Sem essas chaves, **upload de mídia em produção está silenciosamente quebrado**
(o backend cai em fallback 503 quando `R2_ACCESS_KEY_ID`/`R2_SECRET_ACCESS_KEY`
não estão setadas).

**Como criar (≈ 1 minuto):**

1. Cloudflare → R2 → **Manage R2 API Tokens** → **Create Token**.
2. Permission: **Object Read and Write**.
3. Specify bucket: **faren-media** (escopo mínimo).
4. TTL: forever.
5. Copie **Access Key ID** (32 chars) e **Secret Access Key** (64 chars).

**Onde colar (já tem placeholder no `render.yaml`):**

Render dashboard → `faren-api` → Environment → Add:
- `R2_ACCESS_KEY_ID` = ...
- `R2_SECRET_ACCESS_KEY` = ...

(Ou via API:
`curl -X PUT -H "Authorization: Bearer $RENDER_API_KEY" https://api.render.com/v1/services/srv-d7gjdc5ckfvc73ftk79g/env-vars/R2_ACCESS_KEY_ID --data '{"value":"..."}'`)

Após colar, o Render redeploya sozinho.
